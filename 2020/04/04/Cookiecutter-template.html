Cookiecutter

A concise tutorial on cookiecutter for package template.

Apr 4, 2020 â€¢ 2 min read

Its good to have a consistent structure for all the projects/packages. Its better to create a template and reuse it for all projects.adfaadCookiecutter is a great tool for doing this. Let's assume we want to have the following structure.

First install the cookiescutter package as

!pip install cookiecuttter

To define a template, we need two things:

  • Folder/file structure
  • a cookiecutter.json file that acts as a configure file.

The is a template for the project folder with a project name 'project'. Let's define the properties of the project. For that, we need to edit the cookiecutter.json file

Add the follow text to the coookiecutter.json file: This provides good defaults for the project.

{ 
    "project": 'project', 
    'author': 'author', 
    'version': '0.1',
    'description': 'description'
}

Inside the folder, create the following folder and files. We can use the src folder for storing the source files.

We can edit the README.md file as:

## readme of the {{cookiecutter.project}}

Author : {{cookiecutter.author}}
Version: {{cookiecutter.version}}

Similarly, let's create a very basic setup.py file as follow:

import setuptools

setuptools.setup(
    name="{{cookiecutter.project}}", 
    author="{{cookiecutter.author}}", 
    version="{{cookiecutter.version}}",
    classifiers=[
        "Programming Language :: Python :: 3",
        "License :: OSI Approved :: MIT License",
        "Operating System :: OS Independent",
    ],
    python_requires='>=3.6',

)

We are all set, go back to the root folder, where we have the cookiecutter.json file. To create a new package/project based on the template, we can run the following command.

cookiecutter  .

when you run the command, we will be asked to provide the project name, author and the project version, the default are also listed (in the []). If you don't want to provide certain property simplly hit ENTER. Here is an example run.

you will see a new folder created called _testproject

We can now check the README.md and the setup.py files from the newly created project.

## readme of the test_project Author : jnfarooq Version: 0.0.1

and ...

import setuptools

setuptools.setup(
    name="test_project", 
    author="jnfarooq", 
    version="0.0.1",
    classifiers=[
        "Programming Language :: Python :: 3",
        "License :: OSI Approved :: MIT License",
        "Operating System :: OS Independent",
    ],
    python_requires='>=3.6',

)

You can use this as template for all your projects.

Pushing the template to github

One easy way is to push your cookiecutter template to your github and whenever its needed, just pull the repo and use it create new projects. Let's do that. Go to your github account and create a new repo called: cookiecutter

Go to the root folder where the template is and run the following git commands.

!echo "# cookiecutter" >> README.md
!git init
!git add
!git commit -m "first commit"
!git remote add origin https://github.com/username/cookiecutter.git
!git push -u origin master

make sure you replace the username with your github username.

Use it directly from github.

creat a new folder and run the follow commnad:

!cookiecutter https://github.com/username/cookiecutter.git

and it will ask you for all the information listed in the cookiecutter.json file. That's all for a simple template.
